Helix Editor Cheat Sheet

One-Hand (Left) Modal Editing for Rust, JS, Python

Modal editor like Vim - focused on normal mode for efficient coding

Modal Editing: Different modes for different tasks - this cheat sheet focuses on Normal mode commands
Key Principle
Helix uses modal editing: Normal mode for navigation and commands, Insert mode for typing text. Most powerful commands are in Normal mode.

Navigation & Movement

Moving through code efficiently

Action Keys Tip
Move left / right h / l Classic vim-style
Move up / down k / j
Word left / right b / w Jump by words
Line start / end 0 / $
Scroll up / down Ctrl + u / Ctrl + d Half-page scroll
Jump to file top / bottom gg / G Fast travel
Search / then type n / N to cycle results
Movement Pattern
Use hjkl for precise character movement, wb for word movement, and gg/G for large jumps.

Editing & Selection

Making changes to code

Action Keys Notes
Enter insert mode i Type code Insert
Insert at line start / end I / A Handy for editing
Delete char under cursor x
Delete word dw
Delete line dd
Copy (yank) line yy
Paste below / above p / P
Undo / Redo u / Ctrl + r
Select word viw v starts selection
Comment selection gc Works in Rust, JS, etc.
Editing Workflow
Navigate in Normal mode → Use commands like dd, yy, p → Enter Insert mode with i when you need to type → Press Esc to return to Normal mode.

Multi-Cursor & Structural Editing

Helix's powerful editing features

Action Keys Notes
Add cursor below / above Alt + j / Alt + k Works with Sticky Keys
Select next occurrence Ctrl + n Multi-edit repeats
Select all occurrences Ctrl + a
Multi-Cursor Power
Multi-cursor editing lets you make the same change in multiple places simultaneously. Perfect for renaming variables, adding prefixes/suffixes, or bulk formatting.
Example Workflow
To rename a variable:
  1. Place cursor on variable
  2. Ctrl + n to select next occurrence
  3. Repeat until all are selected
  4. Type the new name - it updates all cursors

File & Project Control

Managing files and workspace

Action Keys Notes
Open file picker Space + f (Default Helix "leader" is Space)
Save file :w
Quit :q :wq to save + quit
Split window :vs / :sp Vertical / horizontal
Switch window Ctrl + w then arrow
Toggle file tree (if plugin) Space + e Depends on config
Space as Leader Key
In Helix, Space acts as a "leader" key that brings up additional command menus. Press Space then wait to see available options for file operations, git, etc.

Pro Tips for One-Hand Use

Optimizing Helix for left-hand workflow

Keyboard Remapping

Remap seldom-used keys closer to the left side. In ~/.config/helix/config.toml, you can rebind commands under [keys.normal].

Example remapping:

[keys.normal] space = "file_picker" # Keep default "{" = "goto_previous_buffer" # Remap for easier access "}" = "goto_next_buffer"
Modifier Key Optimization
  • If Ctrl or Alt combos are awkward, consider mapping them to Caps Lock or Backtick
  • Practice "tap sequences" instead of holds — e.g. Ctrl → r (redo) instead of pressing both
  • Use Sticky Keys for multi-key combinations
Essential Workflow Tips
  • Stay in Normal mode as much as possible - it's where the power is
  • Use Esc religiously to return to Normal mode
  • Combine motions with commands: dw (delete word), c$ (change to end of line)
  • Learn to select first (v), then operate on selection
Practice Strategy: Focus on learning 5-10 essential commands first (navigation, basic editing), then gradually add multi-cursor and advanced features.
